Social Media Widget Manager

Mozilla Firefox must be used in order for this application to work!

A web application that is used with the Social Media Widget.

It allows the media.txt file to easily be created/edited without any knowledge of the file specification.

Note: A C++ application is currently in development that has the same functionality as this application.


Additional Information

How to Use the Manager

The Menu dropdown has everything you need in order to create your entries. It is located in the upper right corner of the navigation toolbar.

  • Add an Entry

    To add an entry, select this option. You must select the social media type, along with the title to be displayed, the URL to the page and the Metadata (which are custom categories for the widget). You can create metadata by selecting the "Manage Categories" option in the menu. More than one piece of metadata can be assigned to the entry.

  • Manage Categories

    To add metadata/categories to the entries, you can create a category. This allows you to type in a category name and add it to the category list. You can also edit other categories by selecting this option.

  • Anchors

    These allow easy access to the different types of social media that are in the manager if the list gets too large. They link to the respective social media type.

  • Show Widget

    Shows a preview of the widget and how it will look. A way to see changes as soon as they have been made is in the works, but for now, you must export the file and save it in the same directory as manager.html

  • Export to File...

    This will export all changes that have been made the media.txt file. A download prompt will allow you to save the file. This file should then replace the media.txt on your site and also the media.txt in your manager.html directory.

Uploading media.txt to Your Server

This file is the core of the data that is used in the widget and needs to replace the old media.txt file in order to make changes to the widget.

Simply uploading this file and replacing it on your web server will make all of the changes to the widget. No further modifcations are needed.

Items In Progress and Things to Consider

Currently, the manager and widget work great from my experience. However, there is some code that needs to be updated to make the application more user friendly and easier to use. The following list displays items that I would like to implement in the future (ordered from most important to least important):

  • Displaying a warning message to the user that warns them to export to file when attempting to refresh the page (you will lose data recently entered into the manager if you attempt to refresh the page).

  • Downloading the media.txt file from the server instead of a local directory for easier access and less dependacies.This would also allow other browsers to use the widget.

  • Allowing the Show Widget menu option to display real-time updates.

  • Optimization
